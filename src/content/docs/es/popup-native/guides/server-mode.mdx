# Server Mode

`Mode.Server` es el flujo principal del repositorio.

## Que hace

- Lee `popupOptions.publicKey`
- Descarga popups desde backend
- Evalua triggers, segmentos y cooldowns en cliente
- Renderiza el popup y la encuesta

## Entorno backend

La URL base se deriva de `InitOptions.environment`:

- `Environment.Production` (por defecto): `https://api.deepdots.com`
- `Environment.Development`: `https://api-dev.deepdots.com`

`InitOptions.debug` es independiente y solo controla los logs del SDK. Para hablar
con producción manteniendo los logs activos, deja `environment` en su valor por
defecto y pon `debug = true`.

## Triggers soportados

- `TimeOnPage`
- `Scroll`
- `Exit`
- `Event`
- `Click`
